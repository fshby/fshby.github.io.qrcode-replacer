## 一、项目概述

### 1.1 项目背景
基于 Vue 3 + Vite + Tailwind CSS + Fabric.js 构建的二维码替换工具，核心功能包括：
- 模板图片上传与管理（支持自定义上传和预制模板）
- 二维码检测与角点定位（支持自动检测和手动调整）
- 透视变换与图像合成（支持重新检测）
- 二维码内容实时编辑
- 捐赠入口与开发者联系方式

### 1.2 目标用户
- 营销人员：快速制作带二维码的海报、邀请函
- 设计师：批量生成带不同二维码的物料
- 企业用户：个性化二维码名片制作

## 二、项目周期规划

### 2.1 短期规划（MVP 阶段）

#### 阶段目标
- 快速上线验证核心功能
- 覆盖移动端和小程序场景
- 最小化开发成本，快速迭代
- 建立用户支持渠道（捐赠+联系）

#### 技术方案

**架构选型：纯 Web 响应式 H5 + 小程序 web-view 壳**

| 层级 | 技术方案 | 说明 |
|------|----------|------|
| 前端框架 | Vue 3 + Vite | 现有技术栈，无需改动 |
| 样式方案 | Tailwind CSS 3 | 响应式断点适配 |
| 画布交互 | Fabric.js | 图像编辑、锚点拖拽 |
| 小程序壳 | 原生小程序 + web-view | 快速封装 |
| 通知组件 | Toast | 非阻塞式用户反馈 |

**响应式布局适配**

| 断点 | 设备类型 | 布局策略 |
|------|----------|----------|
| sm（< 640px） | 手机竖屏 | 单列布局，侧边栏折叠为底部抽屉 |
| md（640px - 1024px） | 平板/手机横屏 | 双栏布局，侧边栏收窄 |
| lg（> 1024px） | 桌面端 | 标准双栏布局 |

**小程序 web-view 配置**

```xml
<!-- pages/index/index.wxml -->
<web-view 
  src="https://your-domain.com/qrcode-replacer" 
  bindmessage="onMessage"
></web-view>
```

**业务域名配置要求**

| 配置项 | 说明 |

| 业务域名 | 需在微信公众平台配置（开发管理 → 开发设置） |
| 安全域名 | 图片 CDN 域名需加入白名单 |
| HTTPS | 必须使用 HTTPS 协议 |

### 2.2 中期规划（产品稳定后）

#### 阶段目标
- 根据用户反馈优化产品体验
- 评估是否需要原生能力支持
- 决定技术架构演进方向
- 建立用户增长和变现策略

#### 技术方案评估

**方案对比：继续 H5 vs 迁移 uni-app**

| 维度 | 继续 H5 + web-view | 迁移 uni-app |
|------|-------------------|--------------|
| 开发成本 | 低（无需重构） | 高（Canvas API 重构） |
| 原生能力 | 受限（无法使用支付/分享） | 完整（原生 API 支持） |
| 用户体验 | 接近原生（web-view 性能） | 更好（原生渲染） |
| 跨平台 | 一套代码多端运行 | 一套代码多端运行 |
| 社区支持 | Vue 生态成熟 | uni-app 社区活跃 |

**迁移触发条件**

当满足以下任一条件时，评估迁移方案：

1. 小程序端用户占比超过 50%
2. 用户反馈需要支付、分享等原生能力
3. 需要使用蓝牙、相机等硬件 API
4. web-view 性能无法满足用户体验要求

**Canvas API 迁移策略**

```
Fabric.js 功能 → uni-app Canvas API 替代方案
─────────────────────────────────────────────
图像加载       → uni.getImageInfo
画布初始化     → uni.createCanvasContext
图像绘制       → ctx.drawImage
矩形绘制       → ctx.fillRect / ctx.strokeRect
线条绘制       → ctx.beginPath + ctx.lineTo
触摸事件       → touchstart/touchmove/touchend
图像导出       → uni.canvasToTempFilePath
```

**第三方库推荐**

| 功能 | 推荐库 | 说明 |
|------|--------|------|
| Canvas 交互 | lime-canvas | 类似 Fabric.js 的 Canvas 交互库 |
| 图像压缩 | image-compressor | 前端图像压缩 |
| 二维码生成 | qrcode | 已有依赖，可直接使用 |
| 二维码识别 | jsQR | 已有依赖，可直接使用 |
