## 一、运维概述

### 1.1 项目简介

项目名称：二维码替换工具  
项目类型：纯前端静态应用  
技术栈：Vue 3 + TypeScript + Vite + Tailwind CSS + Fabric.js + ml-matrix  
部署方式：静态文件托管  
核心功能：上传模板图/选择预制模板、二维码检测、透视变换合成、导出PNG、捐赠入口

### 1.2 运维目标

- 确保服务高可用性（99.9%以上）
- 快速响应和解决线上问题
- 定期监控和优化性能
- 保障数据安全和隐私
- 支持小程序 web-view 接入

### 1.3 运维范围

| 范围 | 说明 |
|------|------|
| 部署运维 | 代码构建、部署、版本管理 |
| 性能监控 | 页面加载、资源使用、响应时间 |
| 错误追踪 | JS错误、异常堆栈、用户反馈 |
| 安全运维 | HTTPS配置、XSS防护、CORS配置 |
| 数据管理 | 静态资源管理、模板管理、收款码替换 |
| 小程序运维 | web-view配置、域名审核、版本同步 |

### 2.3 部署流程

#### 2.3.1 手动部署步骤

```bash
# 1. 拉取最新代码
git pull origin main

# 2. 安装依赖（首次部署或依赖变更时）
npm install

# 3. 执行构建（包含TypeScript类型检查）
npm run build

# 4. 部署到静态托管平台
# Vercel: 使用vercel CLI或Git自动部署
# GitHub Pages: 将dist目录推送到gh-pages分支
# 腾讯云COS: 使用cos-cli上传dist目录
```

#### 2.3.2 CI/CD自动部署

**GitHub Actions 配置示例**

```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches: [main]

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
      - name: Install dependencies
        run: npm install
      - name: Build
        run: npm run build
      - name: Deploy
        uses: peaceiris/actions-gh-pages@v4
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./dist
```

## 四、错误追踪与故障排查

### 4.1 常见错误类型

| 错误类型 | 触发条件 | 表现 | 排查方法 |
|----------|----------|------|----------|
| 图片加载失败 | 网络异常、图片URL无效 | 上传面板显示错误 | 检查网络、验证图片URL |
| 二维码检测失败 | 图片模糊、二维码不完整 | Toast提示"未检测到二维码" | 检查图片质量、手动标记锚点 |
| 合成失败 | 透视变换计算出错 | 画布无变化或显示错误 | 检查锚点坐标、验证单应性矩阵 |
| 导出失败 | Canvas污染、浏览器限制 | 下载无响应 | 检查图片跨域设置、更换浏览器 |
| 内存溢出 | 处理超大图片 | 浏览器崩溃或卡顿 | 限制图片尺寸、使用OffscreenCanvas |
| 模板加载失败 | templates.json格式错误 | 预制模板列表为空 | 检查JSON格式、验证图片路径 |

### 4.2 故障排查流程

```
用户反馈问题
    │
    ▼
检查监控系统（Sentry/GitHub Issues）
    │
    ├── 有错误日志 → 分析错误堆栈 → 定位问题代码 → 修复 → 发布
    │
    └── 无错误日志 → 复现问题 → 检查浏览器控制台 → 定位问题 → 修复 → 发布
```

### 4.3 紧急修复流程

```
发现严重问题
    │
    ▼
评估影响范围
    │
    ├── 影响核心功能 → 立即回滚到上一稳定版本 → 修复问题 → 重新发布
    │
    └── 影响次要功能 → 创建Issue → 排期修复 → 发布更新
```

### 4.4 日志分析

前端应用通过console输出关键调试信息：

| 日志类型 | 内容 | 定位方式 |
|----------|------|----------|
| 单应性矩阵验证 | 预测坐标、实际坐标、误差 | 透视变换失败时查看 |
| 逆变换验证 | 源坐标预测、误差 | 图像合成失败时查看 |
| 目标区域 | minX/maxX/minY/maxY | 合成区域异常时查看 |

#### 6.3.2 替换步骤

1. 将真实收款码保存为以下文件名：
   - 微信支付：`wechat-pay.png`
   - 支付宝：`alipay.png`
   - 微信联系：`contact-qr.png`

2. 上传到服务器的 `public/donate/` 目录

3. 清除CDN缓存（如需）

4. 测试扫码功能正常

#### 6.3.3 注意事项

- 收款码图片建议尺寸：300×300px ~ 500×500px
- 使用PNG格式，确保清晰度
- 替换前备份原文件
- 替换后测试扫码是否正常
- 定期更换收款码，确保安全性

#### 7.1.2 web-view页面配置

```xml
<!-- pages/index/index.wxml -->
<web-view 
  src="https://your-domain.com/qrcode-replacer" 
  bindmessage="onMessage"
></web-view>
```

```javascript
// pages/index/index.js
Page({
  onMessage(e) {
    console.log('收到H5消息:', e.detail.data)
  }

```

### 7.2 web-view限制说明

| 限制类型 | 具体内容 | 影响 | 应对策略 |
|----------|----------|------|----------|
| 支付能力 | 无法使用微信支付 | 捐赠功能受限 | 使用扫码支付（捐赠模块已支持） |
| 分享能力 | 无法调用分享API | 传播受限 | 使用H5内部分享链接 |
| 存储能力 | 本地存储受限 | 历史记录受限 | 使用服务端存储（如需） |
| 文件上传 | 受限于web-view | 上传体验 | 使用前端压缩上传 |
| Canvas性能 | 渲染性能受限 | 图像处理卡顿 | 优化图像尺寸，使用离屏Canvas |

### 11.4 关键算法说明

#### 单应性矩阵计算

使用ml-matrix库通过SVD分解计算4点到4点的投影矩阵：

```typescript
// 核心流程
// 1. 构建8×9矩阵A（4对点，每对点生成2行）
// 2. 对A进行SVD分解：A = UΣV^T
// 3. 取V的最后一列作为单应性矩阵H
// 4. 归一化处理（H[8]作为缩放因子）
```

#### 双线性插值

用于透视变换后的像素颜色计算，实现亚像素级精度：

```typescript
// 核心流程
// 1. 获取目标点周围4个像素的坐标
// 2. 计算权重（基于距离的线性插值）
// 3. 加权平均4个像素的颜色值
// 4. 返回插值后的RGBA颜色
```
