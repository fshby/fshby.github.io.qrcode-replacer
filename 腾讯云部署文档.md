## 一、方案概述

### 1.1 架构选择

针对本项目（纯前端静态应用），选择最具性价比的部署方案：

```
用户请求
    │
    ▼
CDN边缘节点（全球加速）
    │
    ▼
COS存储桶（静态网站托管）
    │
    ▼
GitHub Actions（自动构建部署）
```

### 1.2 费用估算

| 服务 | 免费额度 | 超出后计费 | 预估费用（月） |
|------|----------|------------|----------------|
| **COS存储** | 50GB | 0.099元/GB/月 | 约0元（本项目<1GB） |
| **COS外网下行流量** | 10GB/月 | 0.8元/GB | 约0元（小流量） |
| **CDN流量** | 新用户10GB/月 | 0.26-0.5元/GB | 约0-50元 |
| **CDN回源流量** | 无 | 0.2元/GB | 约0-10元 |
| **SSL证书** | 免费DV证书 | - | 0元 |
| **DNS解析** | 免费 | - | 0元 |
| **域名备案** | 免费 | - | 0元 |

**年度预估总费用：500元以内**（主要是CDN流量费）

### 1.3 省钱技巧

- ✅ 使用COS免费额度，存储和流量基本够用
- ✅ 购买CDN流量包（100GB/年约99元），比按量计费更便宜
- ✅ 使用腾讯云免费SSL证书（无需花钱买商业证书）
- ✅ 开启CDN缓存，减少回源流量
- ✅ 图片使用webp格式，减少传输大小

## 三、COS存储桶配置

### 3.1 创建存储桶

1. 登录 [腾讯云COS控制台](https://console.cloud.tencent.com/cos5)
2. 点击「创建存储桶」
3. 填写配置：

| 配置项 | 值 | 说明 |
|--------|-----|------|
| 名称 | `qr-replacer-<随机后缀>` | 如 `qr-replacer-12345` |
| 地域 | 选择靠近用户的地域 | 如「广州」「上海」「北京」 |
| 访问权限 | **公有读私有写** | 静态网站必须公有读，否则返回403 |
| 存储类型 | 标准存储 | 低频访问可用「低频存储」更便宜 |

4. 点击「确定」创建

### 3.2 开启静态网站托管

1. 在存储桶概览页，点击左侧「基础配置」
2. 找到「静态网站」配置项，点击「编辑」
3. 开启静态网站：

| 配置项 | 值 | 说明 |
|--------|-----|------|
| 状态 | 开启 | 启用静态网站托管功能 |
| 索引文档 | `index.html` | 默认首页 |
| 错误文档 | `index.html` | SPA应用需要（路由回退） |

4. 点击「保存」

### 3.3 配置CORS

1. 在存储桶概览页，点击左侧「安全管理」→「跨域访问CORS」
2. 点击「添加规则」：

| 配置项 | 值 | 说明 |
|--------|-----|------|
| 来源Origin | `*` 或具体域名 | 开发环境可先用`*` |
| 允许Methods | `GET, HEAD, OPTIONS` | 静态资源只需这三种 |
| 允许Headers | `*` | 允许所有请求头 |
| 暴露Headers | 留空 | - |
| 缓存时间 | `3600` | 1小时 |

3. 点击「确定」保存

### 3.4 配置防盗链（可选）

1. 在「安全管理」→「防盗链」中开启
2. 添加允许的域名：
   - `https://yourdomain.com`
   - CDN加速域名

## 四、CDN加速配置

### 4.1 添加CDN加速域名

1. 登录 [腾讯云CDN控制台](https://console.cloud.tencent.com/cdn)
2. 点击「添加域名」
3. 填写配置：

| 配置项 | 值 | 说明 |
|--------|-----|------|
| 加速域名 | `qr.yourdomain.com` | 自定义二级域名 |
| 业务类型 | 静态网站 | - |
| 源站类型 | 对象存储COS | - |
| 源站地址 | 选择已创建的COS存储桶 | 自动填充 |
| 回源协议 | HTTPS | 更安全 |

4. 点击「下一步」

### 4.2 配置缓存策略

1. 在CDN域名配置页，点击「缓存规则」
2. 添加以下规则：

| 文件类型 | 缓存时间 | 说明 |
|----------|----------|------|
| `index.html` | 0秒（不缓存） | 确保更新能立即生效 |
| `*.css`, `*.js` | 30天 | 带hash的静态资源 |
| `*.png`, `*.jpg`, `*.webp`, `*.gif` | 30天 | 图片资源 |
| `*.json` | 1小时 | 配置文件 |

3. 点击「保存」

### 4.3 开启HTTPS

1. 在CDN域名配置页，点击「HTTPS配置」
2. 选择「腾讯云SSL证书」→「免费证书」
3. 申请免费SSL证书：
   - 填写域名信息
   - 验证域名所有权（DNS验证或文件验证）
   - 等待审核通过（通常几分钟）
4. 选择已申请的证书，开启HTTPS
5. 开启「强制跳转HTTPS」

### 4.4 配置DNS解析

1. 登录 [腾讯云DNSPod控制台](https://console.dnspod.cn/)
2. 找到你的域名，点击「添加记录」：

| 记录类型 | 主机记录 | 记录值 | TTL |
|----------|----------|--------|-----|
| CNAME | `qr` | CDN提供的CNAME地址 | 5分钟 |

3. 等待DNS生效（通常5-30分钟）

## 五、域名ICP备案

### 5.1 备案必要性

- ✅ 小程序web-view **必须**使用备案域名
- ✅ 使用自定义域名访问必须备案
- ❌ 使用COS默认域名无需备案（但不支持HTTPS）

### 5.2 备案流程

1. 登录 [腾讯云备案控制台](https://console.cloud.tencent.com/beian)
2. 点击「开始备案」
3. 填写备案信息：
   - 主体信息（个人/企业）
   - 网站信息（域名、服务类型等）
4. 提交审核
5. 等待管局审核（通常10-20个工作日）

### 5.3 备案期间临时访问

备案期间无法使用自定义域名，可使用以下临时方案：

```bash
# 使用COS默认域名访问（HTTP）
http://qr-replacer-12345.cos-website.ap-guangzhou.myqcloud.com

# 使用GitHub Pages临时部署（免费HTTPS）
# 在GitHub仓库设置中开启Pages
```

## 六、CI/CD自动部署

### 6.1 配置腾讯云密钥

1. 登录 [腾讯云访问管理控制台](https://console.cloud.tencent.com/cam/capi)
2. 点击「新建密钥」
3. 保存SecretId和SecretKey（只显示一次，务必保存好）

### 6.2 创建GitHub Actions Workflow

在项目根目录创建 `.github/workflows/deploy.yml`：

```yaml
name: Deploy to Tencent Cloud COS

on:
  push:
    branches: [main]

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'

      - name: Install dependencies
        run: npm install

      - name: Build
        run: npm run build

      - name: Install coscli
        run: |
          wget https://github.com/tencentyun/coscli/releases/download/v1.2.22/coscli-linux -O coscli
          chmod +x coscli
          sudo mv coscli /usr/local/bin/

      - name: Configure coscli
        run: |
          coscli config set -a default -b ${{ secrets.COS_BUCKET }} -r ${{ secrets.COS_REGION }}
          coscli config set -a default -sk ${{ secrets.COS_SECRET_KEY }} -ak ${{ secrets.COS_SECRET_ID }}

      - name: Upload to COS
        run: |
          coscli sync ./dist/ cos://${{ secrets.COS_BUCKET }}/ \
            --delete \
            --include "**/*"

      - name: Install tencentcloud-cli
        run: |
          pip install tencentcloud-sdk-python

      - name: Purge CDN cache
        env:
          TENCENTCLOUD_SECRET_ID: ${{ secrets.COS_SECRET_ID }}
          TENCENTCLOUD_SECRET_KEY: ${{ secrets.COS_SECRET_KEY }}
        run: |
          python -c "
          from tencentcloud.common import credential
          from tencentcloud.common.exception.tencent_cloud_sdk_exception import TencentCloudSDKException
          from tencentcloud.cdn.v20180606 import cdn_client, models
          
          try:
              cred = credential.Credential('${{ secrets.COS_SECRET_ID }}', '${{ secrets.COS_SECRET_KEY }}')
              client = cdn_client.CdnClient(cred, '${{ secrets.COS_REGION }}')
              
              req = models.PurgeUrlsCacheRequest()
              req.Urls = ['https://${{ secrets.CDN_DOMAIN }}/index.html']
              
              resp = client.PurgeUrlsCache(req)
              print('CDN缓存刷新成功:', resp.to_json_string())
          except TencentCloudSDKException as err:
              print('CDN缓存刷新失败:', err)
          "
```

### 6.3 配置GitHub Secrets

在GitHub仓库 → Settings → Secrets → Actions中添加：

| Secret名称 | 值 | 说明 |
|------------|-----|------|
| `COS_SECRET_ID` | 腾讯云SecretId | - |
| `COS_SECRET_KEY` | 腾讯云SecretKey | - |
| `COS_BUCKET` | 存储桶名称 | 如 `qr-replacer-12345` |
| `COS_REGION` | 地域 | 如 `ap-guangzhou` |
| `CDN_DOMAIN` | CDN加速域名 | 如 `qr.yourdomain.com` |

## 十、费用优化建议

### 10.1 存储优化

- 使用「低频存储」存放不常访问的模板图片
- 定期清理不需要的历史版本
- 图片压缩后再上传

### 10.2 CDN优化

- 购买CDN流量包（100GB/年约99元）
- 开启智能压缩（GZIP/Brotli）
- 设置合理的缓存时间，减少回源

### 10.3 流量优化

- 使用webp格式图片（比jpg小30%以上）
- 启用CDN边缘压缩
- 限制单次上传文件大小
