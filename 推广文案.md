## 3. 公众号推文（约 600–800 字，可分段）

### 封面标题

> 做活动海报最烦的事，可能就是"换二维码"了 —— 我写了个小工具来解决它

### 正文

每次做活动海报，最折腾人的可能不是排版，而是"**把模板里那个旧二维码换成新的**"。

你可能会：

- 在 PS 里反复截图、缩放、对齐，结果大小或位置没对上；
- 用截图工具直接覆盖一个标准二维码，但原图里二维码是**斜着拍**或**有透视**的，贴上去很违和；
- 找设计师改源文件，来回几轮，活动都要开始了。

于是我写了一个纯前端的小工具 —— **QR Code Replacer**。

**它的工作流程只有 3 步：**

1. **上传一张模板图**（海报、邀请函、名片、长图都可以）；
2. **上传/输入新二维码** —— 既可以传一张二维码图，也可以直接输入一段链接或文字，工具会自动生成一张"干净"的标准二维码；
3. **一键导出 PNG** —— 工具自动识别旧二维码的四角，用**单应性矩阵 + 透视变换**把新二维码贴到原来的位置，支持任意角度、梯形、旋转、带白边等情形。

**技术上它长这样：**

- 前端：Vue 3 + Vite + TypeScript + Tailwind；
- 交互：Fabric.js 管理画布上的锚点，支持手动微调；
- 二维码：jsQR 识别、`qrcode` 库重新生成；
- 核心算法：`Finder Pattern` 定位 → 四角密度投票 → 用 Hartley 归一化 + 8×8 线性方程组求解单应性矩阵，再做反归一化映射到原图坐标。对轴对齐矩形做到 0 像素误差，对任意凸四边形误差远低于 1 像素。

**更重要的一点**：**所有图像运算都在浏览器本地完成**，图片不会上传到任何服务器，也没有账号体系。打开网页即可用。

👉 **在线体验地址**：https://fshby.github.io/fshby.github.io.qrcode-replacer/
👉 **源码仓库**：https://github.com/fshby/fshby.github.io.qrcode-replacer

如果你在活动海报、社群入群码、邀请链接、名片设计等场景里也遇到过"换二维码"的琐碎麻烦，欢迎体验或反馈到 issue。

## 5. 知乎 / 掘金技术文章版（偏技术分享，约 1000 字）

**标题**：写了一个"换海报二维码"的小工具 —— 顺便聊聊单应性矩阵在前端的落地

**正文**：

做活动海报的同学可能都遇到过这种场景：模板图里有个二维码，它既不是正放的正方形，也不是规则的长方形 —— 而是有点**倾斜**、有点**梯形变形**。想用 PS 把一个新二维码贴回去，既要对齐四角，又要做透视变换，还挺费时间。

于是我写了一个纯前端的小工具 **QR Code Replacer**，让整个流程变成三步：上传模板 → 传/输入新二维码 → 导出成品。

核心是三步：

### 1）定位二维码四角

先用经典的 `Finder Pattern`（二维码三个角的方块）做粗定位，拿到二维码中心和大致尺寸；再在四个角附近做局部的密度投票，拟合出一个四边形。然后做"形状健康度"判断——比如四个角不能过度接近一条直线，防止退化情形。

### 2）重新生成一个标准二维码

用户可以上传一张二维码图片（工具会读它的内容），也可以直接输入文字或链接。工具统一重新生成一张干净的标准二维码，保证在最终合成的图里，二维码依然可扫。

### 3）用单应性矩阵做透视合成

把新二维码的四角（标准正方形）当作源点集，把检测到的模板四角当作目标点集，求解单应性矩阵 `H`：

```
p_dst ≈ H · p_src
```

这里用的是 **Hartley 归一化 + 固定 h₉=1 的 8×8 线性方程组**，高斯-约当消元 + 部分主元，再用 `T⁻¹ · H_norm · T` 反归一化。对一般的轴对齐矩形可以做到 0 像素误差，对任意凸四边形误差远低于 1 像素。

最后把新二维码贴到离屏 Canvas 上，再更新到 Fabric 画布，让用户可以继续微调锚点，确认后导出 PNG。

**体验地址**：https://fshby.github.io/fshby.github.io.qrcode-replacer/
**源码**：https://github.com/fshby/fshby.github.io.qrcode-replacer
